# 工作流保存和加载功能改进

## 功能概述

改进后的保存和加载功能提供完整的工作流管理能力：

### 1. URL显示工作流ID
- 保存工作流后，URL自动更新为 `?workflowId=xxx`
- 可以直接通过URL分享或访问特定工作流
- 刷新页面后自动加载URL中指定的工作流

### 2. 加载工作流功能
- 点击"加载"按钮打开工作流列表对话框
- 显示所有已保存的工作流
- 每个工作流显示名称和创建时间
- 按创建时间倒序排列（最新的在前）
- 点击"加载"按钮跳转到对应工作流

## 使用方法

### 保存工作流
1. 在顶部工具栏输入工作流名称
2. 点击"💾 保存"按钮
3. 在弹出对话框中确认名称
4. 保存成功后，URL会自动更新为 `?workflowId=xxx`

### 加载工作流
1. 点击顶部工具栏的"📂 加载"按钮
2. 在弹出对话框中查看所有已保存的工作流
3. 找到要加载的工作流，点击"加载"按钮
4. 页面会跳转到对应的工作流，URL也会更新

### 通过URL访问
- 直接访问带有 `?workflowId=xxx` 参数的URL
- 系统会自动加载指定的工作流
- 例如：`http://localhost:5174/?workflowId=abc123`

## 技术实现

### 文件结构
```
frontend/src/
├── App.tsx                              # URL参数处理和自动加载
├── components/
│   ├── layout/TopBar.tsx               # 保存和加载按钮
│   └── modals/
│       ├── SaveWorkflowModal.tsx       # 保存对话框
│       └── LoadWorkflowModal.tsx       # 加载对话框（新增）
└── api/workflowApi.ts                  # 后端API调用
```

### URL参数处理
```typescript
// App.tsx
const urlParams = new URLSearchParams(window.location.search);
const workflowIdFromUrl = urlParams.get('workflowId');

if (workflowIdFromUrl) {
  // 加载指定工作流
  const workflow = await workflowApi.get(workflowIdFromUrl);
  // ...
}
```

### 保存后更新URL
```typescript
// TopBar.tsx
const result = await workflowApi.create({ name, definition });
setWorkflowId(result.id);
// 更新URL
window.history.pushState(null, '', `?workflowId=${result.id}`);
```

### 加载工作流
```typescript
// TopBar.tsx
const handleLoadWorkflow = (loadedWorkflowId: string) => {
  // 跳转到新URL，触发页面重新加载
  window.location.href = `?workflowId=${loadedWorkflowId}`;
};
```

## 功能特点

### URL持久化
- 工作流ID存储在URL中，便于分享
- 刷新页面不会丢失当前工作流
- 支持浏览器前进/后退按钮

### 用户体验
- 工作流列表按时间排序，最新在前
- 鼠标悬停高亮显示
- 清晰的视觉反馈
- 错误提示友好

### 数据安全
- 保存前验证工作流名称
- 加载失败时显示错误信息
- 网络异常时给出提示

## 注意事项

1. **后端服务**：需要后端服务正常运行才能保存和加载工作流
2. **URL分享**：可以通过URL分享工作流给其他用户
3. **浏览器历史**：加载工作流会添加浏览器历史记录
4. **自动加载**：如果URL中有工作流ID，会优先加载该工作流

## 使用场景

### 场景1：创建新工作流
1. 访问应用首页（无URL参数）
2. 编辑工作流
3. 保存后URL自动更新

### 场景2：继续编辑现有工作流
1. 点击"加载"按钮
2. 从列表中选择工作流
3. 继续编辑和保存

### 场景3：分享工作流
1. 保存工作流后复制URL
2. 发送给其他用户
3. 其他用户打开URL即可看到相同工作流

### 场景4：书签收藏
1. 保存常用工作流
2. 将URL添加到浏览器书签
3. 通过书签快速访问
