# 调试功能改进说明

## 功能概述

调试功能已全面改进，现在包含三个主要视图：

### 1. 执行状态视图
显示每个节点的执行状态和时间线。

**功能特点：**
- 可视化的时间线进度
- 节点状态实时更新（等待中、执行中、已完成、失败）
- 执行时间统计
- 可展开查看输出数据
- 失败时显示错误信息

**状态指示：**
- 🔵 蓝色 - 执行中（带动画效果）
- 🟢 绿色 - 已完成
- 🔴 红色 - 失败
- ⚪ 灰色 - 等待中/未执行

### 2. 执行日志视图
记录完整的执行过程日志。

**日志类型：**
- ℹ️ 信息 - 执行开始、节点启动等
- ✅ 成功 - 节点完成、工作流完成
- ❌ 错误 - 执行失败、节点错误
- ⚠️ 警告 - 潜在问题提醒
- 📊 数据 - 数据流转记录

**日志内容：**
- 精确的时间戳（精确到毫秒）
- 事件消息
- 关联的节点信息
- 详细的数据内容（可展开查看）

**特性：**
- 自动滚动到最新日志
- 日志计数显示
- 按类型颜色编码

### 3. 数据流转视图
可视化显示数据在节点间的流转过程。

**显示内容：**
- 节点执行顺序
- 每个节点的输入数据
- 每个节点的输出数据
- 数据流转方向指示
- 执行时间统计
- 错误信息展示

**交互功能：**
- 点击节点卡片展开详细数据
- 输入数据（绿色背景）和输出数据（蓝色背景）分开展示
- 错误信息高亮显示（红色背景）

## 使用方法

1. 点击顶部工具栏的"调试"按钮打开调试面板
2. 在输入框中输入测试数据
3. 点击"运行"按钮开始执行
4. 通过三个选项卡查看不同维度的调试信息：
   - **执行状态**：查看整体进度和节点状态
   - **执行日志**：查看详细的时间线日志
   - **数据流转**：查看数据如何在节点间传递

## 数据结构

### ExecutionLog
```typescript
interface ExecutionLog {
  id: string;
  timestamp: number;
  type: 'info' | 'success' | 'error' | 'warning' | 'data';
  message: string;
  nodeId?: string;
  nodeName?: string;
  data?: unknown;
}
```

## 调试技巧

1. **快速定位问题**：查看执行日志中的错误类型日志
2. **验证数据流**：在数据流转视图中检查每个节点的输入输出
3. **性能分析**：查看每个节点的执行时间，找出性能瓶颈
4. **完整追踪**：结合三个视图，全面了解执行过程
