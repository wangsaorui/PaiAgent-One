# 工作流保存功能说明

## 功能概述

现已实现完整的工作流保存功能，包括：
- 创建新工作流
- 保存现有工作流
- 工作流命名功能
- 用户友好的操作反馈

## 使用方法

### 1. 保存工作流
1. 在顶部工具栏的输入框中输入工作流名称
2. 点击"保存"按钮
3. 在弹出的对话框中确认工作流名称
4. 点击"保存"或"创建"按钮完成操作

### 2. 功能特点

#### 智能保存逻辑
- **新建工作流**：如果当前没有工作流ID，会创建新的工作流
- **更新工作流**：如果有工作流ID，则更新现有工作流

#### 用户体验优化
- **模态对话框**：提供专注的命名界面
- **状态反馈**：保存过程中显示加载状态
- **成功提示**：保存完成后显示成功消息
- **错误处理**：网络异常时给出友好提示
- **表单验证**：防止空名称保存

#### 界面改进
- 增大的输入框和按钮
- 更清晰的视觉层次
- 键盘支持（Enter提交，ESC取消）
- 自动焦点获取

## 技术实现

### 文件结构
```
src/
├── components/
│   ├── layout/TopBar.tsx          # 顶部工具栏（保存按钮）
│   └── modals/SaveWorkflowModal.tsx  # 保存对话框
├── api/workflowApi.ts             # 后端API调用
└── store/workflowStore.ts         # 状态管理
```

### 核心功能
1. **TopBar组件**：管理工作流名称输入和保存触发
2. **SaveWorkflowModal组件**：提供专业的命名和保存界面
3. **workflowApi**：与后端REST API通信
4. **状态同步**：保存后自动更新本地状态

## API接口

后端提供以下REST接口：
- `POST /api/workflows` - 创建新工作流
- `PUT /api/workflows/{id}` - 更新现有工作流
- `GET /api/workflows` - 获取工作流列表
- `GET /api/workflows/{id}` - 获取特定工作流

## 注意事项

1. 需要后端服务正常运行才能保存
2. 工作流名称不能为空
3. 保存时会序列化整个工作流定义（节点、连线、配置）
4. 成功保存后会在本地存储工作流ID用于后续更新
