# 工作流保存和加载功能完善说明

## 功能概述

完善后的保存和加载功能支持多个工作流的管理，包括：

### 1. 保存多个工作流
- 可以创建多个不同的工作流
- 支持"另存为新工作流"选项
- 显示已保存的工作流列表

### 2. 加载工作流
- 点击"加载"按钮显示所有已保存的工作流列表
- 列表显示工作流名称和创建时间
- 点击"加载"按钮直接加载到当前画布（不刷新页面）

## 使用方法

### 保存工作流

#### 场景1：创建新工作流
1. 编辑工作流
2. 点击"💾 保存"按钮
3. 输入工作流名称
4. 点击"创建"按钮
5. URL自动更新为 `?workflowId=xxx`

#### 场景2：更新现有工作流
1. 打开已保存的工作流
2. 修改工作流
3. 点击"💾 保存"按钮
4. 确保"另存为新工作流"复选框未选中
5. 点击"更新"按钮

#### 场景3：另存为新工作流
1. 打开任意工作流
2. 点击"💾 保存"按钮
3. 勾选"另存为新工作流"复选框
4. 输入新的工作流名称
5. 点击"创建"按钮
6. 创建成功后，URL更新为新工作流的ID

### 加载工作流

1. 点击"📂 加载"按钮
2. 在弹出对话框中查看所有已保存的工作流
3. 工作流按创建时间倒序排列（最新的在前）
4. 找到要加载的工作流，点击"加载"按钮
5. 工作流直接加载到当前画布，URL自动更新

## 技术实现

### 关键改进

#### 1. WorkflowStore新增方法
```typescript
loadWorkflow: (id, name, workflowNodes, workflowEdges) => {
  // 将工作流节点和边转换为ReactFlow格式
  // 直接更新store状态，不刷新页面
}
```

#### 2. 加载工作流不刷新页面
```typescript
// 旧实现（会刷新页面）
window.location.href = `?workflowId=${loadedWorkflowId}`;

// 新实现（不刷新页面）
const workflow = await workflowApi.get(loadedWorkflowId);
loadWorkflow(workflow.id, workflow.name, def.nodes, def.edges);
window.history.pushState(null, '', `?workflowId=${loadedWorkflowId}`);
```

#### 3. 保存时支持另存为新工作流
```typescript
// SaveWorkflowModal新增参数
interface SaveWorkflowModalProps {
  onSave: (name: string, asNew: boolean) => void;
  workflowId: string | null;
}

// 保存逻辑
if (saveAsNew || !workflowId) {
  // 创建新工作流
  const result = await workflowApi.create({ name, definition });
} else {
  // 更新现有工作流
  await workflowApi.update(workflowId, { name, definition });
}
```

## 功能特点

### 多工作流管理
- ✅ 创建多个独立的工作流
- ✅ 每个工作流有唯一的ID和名称
- ✅ 按时间排序显示所有工作流

### 用户体验
- ✅ 加载工作流不刷新页面
- ✅ 画布直接更新，无需等待
- ✅ URL自动同步更新
- ✅ 清晰的保存选项（更新/另存为）

### 数据安全
- ✅ 保存前验证工作流名称
- ✅ 显示已存在的工作流列表
- ✅ 防止意外覆盖

## 使用场景

### 场景1：管理多个项目
- 为不同项目创建不同的工作流
- 通过加载功能快速切换

### 场景2：版本管理
- 修改前另存为新版本
- 保留历史版本

### 场景3：模板复用
- 创建模板工作流
- 另存为新工作流进行定制

### 场景4：团队协作
- 每个成员创建自己的工作流
- 通过URL分享特定工作流

## 注意事项

1. **后端服务**：需要后端服务正常运行
2. **工作流命名**：建议使用有意义的名称便于识别
3. **URL分享**：通过URL分享的工作流可以被其他用户加载
4. **另存为**：勾选"另存为新工作流"会创建新的工作流ID
